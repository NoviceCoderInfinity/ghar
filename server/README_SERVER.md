# Ghar server ‚Äî run: `cd server && pip install fastapi uvicorn google-genai && uvicorn app:app --host 0.0.0.0 --port 8000`
- Env vars (`.env` at repo root works): `GEMINI_API_KEY` (required), `IMAGE_MODEL` (default `gemini-3.1-flash-image`), `BRIEF_MODEL` (default `gemini-3.5-flash`), `TOUR_MODEL` (default `gemini-omni-flash-preview`), `PLAN_MODEL` (default `gemini-3.1-flash-image`).
- `curl -X POST localhost:8000/variants -H 'Content-Type: application/json' -d "{\"description\":\"replace the chair with a rattan armchair\",\"keyframe_b64\":\"$(base64 -i ../demo/fixtures/corner.jpg)\"}"` ‚Üí `{"batch_id":"b_xxx","count":4}` instantly.
- `keyframe_b64` is now OPTIONAL: omit it and each slot generates a from-scratch concept photo of the description (same 4 style suffixes) instead of editing a photo. `curl -X POST localhost:8000/variants -H 'Content-Type: application/json' -d '{"description":"a cozy reading corner with a rattan armchair"}'` ‚Üí same `{"batch_id":"b_xxx","count":4}` + poll shape.
- `curl localhost:8000/variants/b_xxx` ‚Üí per-slot `done/pending/failed` + `/static/` URLs (rail polls this every 1s).
- `POST /tour` ‚Äî Omni Flash image‚Üívideo walkthrough (~40‚Äì120s render). Body: exactly ONE of `image_b64` (raw/data-URL base64 jpeg) or `image_url` (a `/static/...` path served by this server, e.g. a variant output; other URLs rejected). Returns `{"job_id":"t_xxx"}` instantly.
  - `curl -X POST localhost:8000/tour -H 'Content-Type: application/json' -d '{"image_url":"/static/b_xxx_0.jpg"}'`
  - or `curl -X POST localhost:8000/tour -H 'Content-Type: application/json' -d "{\"image_b64\":\"$(base64 -i ../demo/fixtures/corner.jpg)\"}"`
- `POST /tour` also takes an optional `instruction` (extra camera/scene direction appended to the tour prompt):
  - `curl -X POST localhost:8000/tour -H 'Content-Type: application/json' -d '{"image_url":"/static/b_xxx_0.jpg","instruction":"linger on the reading corner, end facing the window"}'`
- `curl localhost:8000/tour/t_xxx` ‚Üí `{"job_id":"t_xxx","status":"pending"|"done"|"failed","video_url":"/static/tour_t_xxx.mp4"|null}` (poll every few seconds; 404 for unknown ids).
- `POST /plan` ‚Äî labelled concept floor-plan image (top-down 2D, "not to scale" title block) for a whole-home description. Returns `{"job_id":"p_xxx"}` instantly; image renders in the background to `static/plan_p_xxx.jpg`.
  - `curl -X POST localhost:8000/plan -H 'Content-Type: application/json' -d '{"home_description":"3BHK, 2 bedrooms, a gym room, underground parking, modern techy vibe"}'` ‚Üí `{"job_id":"p_xxx"}`
  - `curl localhost:8000/plan/p_xxx` ‚Üí `{"job_id":"p_xxx","status":"pending"|"done"|"failed","image_url":"/static/plan_p_xxx.jpg"|null}` (404 for unknown ids).
- `curl -X POST localhost:8000/brief -H 'Content-Type: application/json' -d '{"description":"warm minimal corner with rattan armchair","objects":["armchair","rug","floor lamp"]}'` ‚Üí budget+legal+materials_and_equipment JSON. Demo cache: `cp static/brief_cache_example.json static/brief_cache.json` to make it instant.
- `/brief` whole-home mode ‚Äî add optional `home_description` (report covers the WHOLE home) and `rooms` (adds a `"rooms": [{"room","spec"}]` section with a 2-line design spec per room). Whole-home requests SKIP the room-era `brief_cache.json` and use `brief_cache_v2.json` instead (written on first live success); the `brief_cache_example.json` fallback still applies if the live call fails.
  - `curl -X POST localhost:8000/brief -H 'Content-Type: application/json' -d '{"description":"modern techy 3BHK, matte black + walnut, smart lighting","objects":["sofa","treadmill","study desk"],"home_description":"3BHK, 2 bedrooms, a gym room, underground parking, modern techy vibe","rooms":["living room","master bedroom","bedroom 2","gym room"]}'` ‚Üí budget+legal+rooms+materials_and_equipment JSON.
- Architect build pack = `/plan` image + whole-home `/brief` report (both driven by the same `home_description`).
- `POST /export` ‚Äî turns the pack into a downloadable, print-ready self-contained HTML document (`Content-Disposition: attachment; filename="ghar-architect-pack.html"`). Body: `brief` (the full brief JSON the client already has, required), `plan_image_url` (a `/static/plan_*.jpg` path served by this server, or null ‚Äî section skipped if missing; image is embedded as a base64 data-URI so the file works offline), `home_description` (string or null, shown under the title). Sections: title + date + home description, floor plan, room-by-room specs, budget with total (‚Çπ, "estimate" tags, vendor URLs as plain text), materials & equipment, legal checklist, disclaimer footer.
  - `curl -X POST localhost:8000/export -H 'Content-Type: application/json' -d "{\"brief\":$(cat static/brief_cache_v2.json),\"plan_image_url\":\"/static/plan_p_xxx.jpg\",\"home_description\":\"3BHK, 2 bedrooms, a gym room, modern techy vibe\"}" -o ghar-architect-pack.html`
